What's included

15 commands. Full shipping loop.

Every stage from planning to post-ship. Each command reads your actual codebase and returns structured, actionable output.

/spec
Feature spec from plain English — user stories, acceptance criteria, edge cases, files likely touched.
/scaffold
Describe a component. Get the exact files and starter code to create.
/test-write
Reads your code and writes the tests. Unit, integration, edge cases.
/pr-review
Reviews diff for bugs, security issues, missing tests. Returns MUST FIX / SHOULD FIX / NITPICK.
/commit-msg
Reads staged diff, outputs a conventional commit message. Ready to paste.
/changelog
Generates CHANGELOG entry from recent commits. Semver-ready.
/env-check
Audits .env files vs. code. Finds missing vars, unused vars, no secrets in .env.example.
/db-migrate
Reviews migration files for destructive ops, missing rollbacks, index implications.
/error-triage
Paste a stack trace. Returns root cause, explanation, and the fix.
/deploy-check
Pre-deployment checklist — rate limits, auth, env vars, migrations safe to run.
/api-stub
Describe an endpoint. Get route, handler, validation, and error handling scaffold.
/security-audit
Scans code for common vulnerabilities before they ship.
/docs-update
Reads recent changes and updates relevant documentation.
/refactor
Targeted refactor with rationale. Doesn't change behavior, cleans the code.
/load-test
Generates a load testing script for a new endpoint before prod finds out.
